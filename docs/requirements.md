# X 投稿支援 Chrome 拡張機能 要件定義

## 1. 概要

本書は、閲覧中のページ URL または選択テキストを X に投稿しやすくする
Chrome 拡張機能の要件を整理したものである。

本拡張機能の責務は、X の投稿画面を開き、投稿本文を事前入力した状態を作る
ところまでとする。最終的な投稿実行は常にユーザーが行う。

実装上の具体的な判断は `docs/implementation-decisions.md` を参照する。

## 2. 目的

- 閲覧中のページ URL を、少ない操作で X に共有できるようにする
- ページ内で選択したテキストを引用し、X の投稿画面を開けるようにする
- 拡張機能アイコンから現在のページをすぐに共有できるようにする

## 3. 非目的

初期版では、以下は対象外とする。

- X への自動投稿
- X アカウント認証
- 投稿履歴管理
- URL 短縮
- 拡張機能側での高度な引用編集 UI
- X の投稿画面を置き換える独自確認 UI

## 4. 対象範囲

### 対象

- Chrome 拡張機能
- 拡張機能アイコンからの投稿起動
- 右クリックメニューからの投稿起動
- X 投稿画面の起動と投稿文の事前入力
- 設定画面の提供

### 非対象

- ユーザーの代わりに投稿を実行する処理
- X 側 API 連携が必要な機能

## 5. 想定ユースケース

### UC-1: ページ URL を共有する

1. ユーザーが任意の Web ページを開く
2. ページ上で右クリックする
3. `Share page on X` を選択する
4. 拡張機能が X の投稿画面を開く
5. 投稿本文に現在のページ URL が事前入力される
6. ユーザーが内容を確認し、投稿するか判断する

### UC-2: 選択テキストを引用して共有する

1. ユーザーがページ内テキストの一部を選択する
2. 選択状態で右クリックする
3. `Quote selected text on X` を選択する
4. 拡張機能が X の投稿画面を開く
5. 投稿本文に選択テキストと現在のページ URL が事前入力される
6. ユーザーが内容を確認し、投稿するか判断する

### UC-3: 拡張機能アイコンから現在のページを共有する

1. ユーザーが拡張機能アイコンをクリックする
2. 拡張機能が現在アクティブなタブの URL を取得する
3. 拡張機能が X の投稿画面を開く
4. 投稿本文に現在のページ URL が事前入力される
5. ユーザーが内容を確認し、投稿するか判断する

## 6. 機能要件

### FR-1: 拡張機能アイコン押下時の挙動

- 拡張機能アイコン押下時は、現在アクティブなタブの URL を使って
  ページ共有を開始すること
- 初期版では、アイコンクリック時にアクションポップアップを表示しないこと

### FR-2: ページ右クリックメニュー

- 通常の Web ページ上で右クリックしたとき、
  `Share page on X` を表示すること
- テキスト未選択時でも表示されること

### FR-3: テキスト選択時の右クリックメニュー

- テキスト選択中に右クリックしたとき、
  `Quote selected text on X` を表示すること
- このメニューは、選択テキストが存在する場合のみ表示されること

### FR-4: ページ共有時の動作

- `Share page on X` 実行時、現在アクティブなタブの URL を取得すること
- X に渡す URL は、共有に不要なクエリパラメータを可能な範囲で除去した
  ものを利用すること
- X の投稿画面を別の閲覧コンテキストで開くこと
- 初期版では、popup window で開くこと
- 投稿本文は、ページ共有用テンプレートで事前入力されること

### FR-5: 選択テキスト引用時の動作

- `Quote selected text on X` 実行時、選択テキストと現在アクティブなタブの
  URL を取得すること
- X に渡す URL は、共有に不要なクエリパラメータを可能な範囲で除去した
  ものを利用すること
- 選択テキストは `twitter-text` の weighted length で 150 文字以内に収める
  こと
- 選択テキストが制限を超える場合は、選択テキスト側を末尾 `...` 付きで
  短縮すること
- X の投稿画面を別の閲覧コンテキストで開くこと
- 初期版では、popup window で開くこと
- 投稿本文は、引用共有用テンプレートで事前入力されること

### FR-6: 投稿責務の範囲

- 拡張機能は、X の投稿画面を開くところまでを責務とすること
- 最終的な投稿処理は実行しないこと

### FR-7: 設定画面の内容

設定画面には、最低限以下を含めること。

- 拡張機能の説明
- 利用可能な右クリックメニューの説明
- 投稿テンプレートの説明
- 将来的な設定項目追加に備えた拡張余地

設定画面は、ブラウザが提供する拡張機能の options 導線から開けること。

## 7. データ要件・テンプレート要件

### 入力データ

- 現在表示中ページの URL
- 現在選択中のテキスト

### 出力データ

- X の投稿画面へ渡す投稿本文

### 出力前の加工ルール

- URL は、共有に不要なクエリパラメータを可能な範囲で除去してから利用する
- 選択テキストの文字数計算には `twitter-text` を用いる
- 選択テキストの文字数制限は weighted length 150 とする
- 選択テキストは、制限を超える場合、末尾を `...` にした短縮文字列に
  変換してから利用する

初期版の実装では、以下を具体値として採用する。

- 選択テキストの上限は weighted length 150 とする
- 除去対象クエリパラメータは `fbclid` `gclid` `igshid` `mc_cid` `mc_eid`
  `ref` `si` および `utm_` で始まるものとする

### 投稿テンプレート

#### ページ共有テンプレート

```text
読んでる ${url}
```

#### 選択テキスト引用テンプレート

引用テンプレートは `>` 記法を正とする。

```text


> ${text}
${url}
```

先頭の改行2つはユーザーがそこに入力する想定

## 8. 例外・境界条件

### EC-1: URL を取得できない場合

- 有効な URL を取得できない場合、X の投稿画面を開かないこと
- 実装に応じて、簡易なエラー通知を出してよい

### EC-2: 選択テキストが空の場合

- `Quote selected text on X` は表示しないこと
- 何らかの理由で実行されても、引用投稿処理を継続しないこと

### EC-3: 選択テキストが長すぎる場合

- 選択テキストが weighted length 150 を超える場合は、そのまま渡さず、
  末尾を `...` にした短縮文字列へ変換すること
- 短縮後の文字列を使って投稿本文を組み立てること

### EC-4: URL に不要なクエリパラメータが含まれる場合

- 共有に不要なクエリパラメータは、可能な範囲で除去すること
- ただし、ページ識別に必要なパラメータまで除去しないこと

### EC-5: 特殊文字を含む場合

- 投稿本文は、安全にエンコードして X に渡すこと
- 改行、記号、日本語、その他の非 ASCII 文字を正しく扱えること

### EC-6: サポート対象外ページ

- `chrome://` ページや拡張機能ページなど、通常ページと同様に扱えない
  ページが存在する
- その場合は、機能を無効化するか、エラーとして扱うこと

## 9. 非機能要件

### NFR-1: 操作性

- ユーザーが右クリックメニューから 1 アクションで X の投稿画面まで到達
  できること

### NFR-2: 応答性

- メニュー選択後、体感上すぐに X の投稿画面が開くこと

### NFR-3: 保守性

- 投稿テンプレートを後から追加・変更しやすいこと
- 共有メニューを後から追加しやすいこと

### NFR-4: プラットフォーム制約

- Manifest V3 を前提とすること

### NFR-5: 権限最小化

- 実装に必要な最小限の権限に留めること
- 想定権限は以下とする
  - `contextMenus`
  - `activeTab`
  - `tabs`
  - `storage` は永続設定を導入する場合のみ

### NFR-6: 実装基盤

- 実装基盤には WXT を採用すること
- options UI 実装には React と CSS を用いること
- ディレクトリ構成は WXT の React template に可能な限り揃えること

## 10. 受け入れ条件

### AC-1

拡張機能アイコンをクリックすると、設定画面が開くこと。

### AC-2

通常の Web ページ上で右クリックすると、`Share page on X` が表示される
こと。

### AC-3

ページ内テキストを選択して右クリックすると、`Quote selected text on X` が
表示されること。

### AC-4

`Share page on X` 実行時、X の投稿画面に以下の本文が事前入力されること。

このとき URL には、共有に不要なクエリパラメータが含まれないこと。

```text
読んでる ${url}
```

### AC-5

`Quote selected text on X` 実行時、X の投稿画面に以下の本文が事前入力される
こと。

このとき URL には、共有に不要なクエリパラメータが含まれないこと。
また、選択テキストが長すぎる場合は、末尾が `...` の短縮文字列に置き換わる
こと。

```text


> ${text}
${url}
```

### AC-6

拡張機能は投稿を自動実行せず、X の投稿画面表示までで停止すること。

### AC-7

長い選択テキストで `Quote selected text on X` を実行した場合、短縮後の本文が
X の投稿画面に事前入力されること。

### AC-8

トラッキング用途などの不要なクエリパラメータを含む URL で共有した場合、
不要パラメータを除去した URL が X の投稿画面に事前入力されること。

### AC-9

`Share page on X` および `Quote selected text on X` 実行時、X の投稿画面は
新規タブで開くこと。

## 11. 実装で確定した事項

- 拡張機能のビルド・manifest 生成・entrypoint 管理には WXT を使用する
- options 画面は React と CSS で実装する
- ディレクトリ構成は WXT の React template に近い構成を採用する
- 拡張機能アイコン押下時は options ページをタブで開く
- X の投稿画面は新規タブで開く
- 引用時の選択テキストの上限は weighted length 150 とする
- URL 正規化では `fbclid` `gclid` `igshid` `mc_cid` `mc_eid` `ref` `si`
  および `utm_` で始まるクエリパラメータを除去対象とする
- 初期版では、エラー時にユーザー向け通知 UI は出さず、投稿画面を開かずに
  拡張機能側ログへ記録する

## 12. 未確定事項

初期版では実装判断に委ねてもよいが、挙動を固定したい場合は以下を別途決定
する。

- テンプレート変更機能を、いつ設定画面に公開するか
- `storage` を導入して永続設定を持たせるか
